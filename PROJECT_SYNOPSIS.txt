PROJECT TOPIC
================================================================================

Real-Time Phishing Detection Using Encrypted Traffic Fingerprinting at 
Pre-Connection Stage

This project develops a production-ready system for detecting phishing websites 
BEFORE full page load by analyzing only network-level features (DNS queries, 
TLS handshakes, and traffic flow patterns) without requiring payload decryption 
or browser access.


OBJECTIVES
================================================================================

1. EARLY DETECTION
   - Identify phishing at DNS/TLS handshake stage before any webpage content
     is transmitted
   - Achieve detection at pre-connection stage using only encrypted traffic

2. NON-INVASIVE APPROACH
   - Implement detection without payload decryption
   - No browser extensions or invasive monitoring required
   - Privacy-preserving analysis of encrypted network traffic

3. REAL-TIME PERFORMANCE
   - Achieve low-latency decision making (<50ms)
   - Implement automated blocking mechanism
   - Support network-level enforcement

4. HIGH ACCURACY
   - Achieve 92.31% accuracy using cross-validation
   - Maintain high precision (90%) to minimize false positives
   - Optimize recall (85%) to catch phishing attempts

5. MODULARITY & EXTENSIBILITY
   - Design modular architecture for easy testing and validation
   - Support multiple ML models for comparison
   - Enable easy integration into network infrastructure

6. COMPLETE PIPELINE
   - Implement end-to-end system from packet capture to blocking decisions
   - Create comprehensive feature engineering pipeline
   - Develop integrated decision engine with logging


BRIEF METHODOLOGY
================================================================================

1. DATA COLLECTION
   - Source: PhishTank API (real-world phishing domains)
   - Collection: 30+ domains per dataset run
   - Total Dataset: 65 unique domains
   - Distribution: 17 phishing samples (26.2%), 48 legitimate (73.8%)
   - Stratified sampling to handle class imbalance

2. FEATURE ENGINEERING (48 Features Total)
   
   DNS Features (11 features):
   - TTL values, query types, response codes
   - Domain entropy, character distribution
   - DNS response time patterns
   
   TLS/Certificate Features (10 features):
   - TLS version information
   - Certificate age and issuer analysis
   - Certificate subject patterns
   
   Traffic Flow Features (13 features):
   - Packet count and timing patterns
   - Port usage patterns
   - Packet size distributions
   - Flow duration and inter-arrival times
   
   Domain Characteristics (14 features):
   - Domain length and entropy
   - Special character counts
   - Hyphen placement patterns
   - Domain registration indicators

   KEY INSIGHT: Entropy-based features effectively differentiate phishing
   domains (entropy >4.5) from legitimate domains (entropy <3.5)

3. MODEL TRAINING & COMPARISON
   - Train/Test Split: 80/20 stratified split (52 train, 13 test samples)
   - 5-fold cross-validation for robust evaluation
   - Models Evaluated:
     * Random Forest (100 trees, max_depth=15) ← SELECTED MODEL
     * Gradient Boosting Classifier
     * Support Vector Machine (SVM)
     * Neural Network (3-layer perceptron)

4. MODEL EVALUATION
   - Random Forest Selected Based On:
     * Test Accuracy: 100%
     * 5-Fold CV Accuracy: 92.31% ± 4.87%
     * F1 Score: 84.14% ± 8.62%
     * ROC-AUC: 89.39% ± 9.93%
     * Zero false positives on test set

5. REAL-TIME INFERENCE
   - Load trained model for live predictions
   - Extract features from network traffic in real-time
   - Generate confidence scores for classification

6. DECISION ENGINE & BLOCKING
   - Decision Threshold: 85% confidence for blocking
   - Alert Threshold: 65-85% confidence for alerts
   - Logging: JSONL format with detection metadata
   - Blocking: DNS/network-level enforcement

CORE MODULES
------------
1. Packet Capture Module (packet_capture.py)
   - Live network sniffing with Scapy
   - Captures DNS and TLS handshake data
   - Threaded background capture support
   - Extracts: source/destination IPs, ports, protocols

2. Feature Engineering Module (feature_engineering.py)
   - Generates 48 features from network data
   - DNS feature extraction (entropy, TTL, response patterns)
   - TLS certificate analysis
   - Traffic flow statistics
   - Domain characteristic analysis
   - Feature normalization and vectorization

3. ML Model Module (ml_model.py)
   - Trains multiple ML models
   - Implements cross-validation evaluation
   - Model persistence (pickle format)
   - Metadata storage (model_metadata.json)
   - Comparison metrics reporting

4. Real-Time Engine (realtime_engine.py)
   - Loads trained model and scaler
   - Performs live inference on network traffic
   - Generates predictions with confidence scores
   - Handles batch and single-sample predictions

5. Decision Engine (decision_engine.py)
   - Processes model predictions
   - Applies blocking thresholds
   - Generates alerts and blocking decisions
   - Logs detection events (JSONL format)
   - Maintains blocked domains list
